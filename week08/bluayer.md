# Chapter 7. API 관리 및 사용 패턴

API 관리 패턴을 활용해 관리형 API로 제공

## API 관리 패턴

비즈니스 기능은 사용자들에게 API 형태로 제공됩니다.

- 외부에 노출하고자 하는 API에 직접 참여할 수 있음
- 애플리케이션을 다른 서비스와 통합해서 새로운 기능을 만들 수 있음
- API 사용 촉진, 비즈니스 분석 결과, 통찰 제공

## API 게이트웨이 패턴

API 게이트웨이 패턴 -> 비즈니스 기능 전면에 별도의 계층을 두어서 사용자에게 서비스를 노출, 제공

파사드 패턴

- 게이트웨이 계층에는 비즈니스 기능 관련 로직을 구현해서는 안 된다.

## API 마이크로게이트웨이 패턴

API 게이트웨이를 분산해서 API 게이트웨이의 각 API를 독립적인 실행 환경으로 배포

각 API가 고유한 실행 환경을 가지고, API 매니지먼트 플레인을 통해 관리

## 서비스 메시 사이드카 게이트웨이 패턴

API 게이트웨이 런타임이 아닌 서비스 메시를 사용하는 환경에서 API 게이트웨이 기능을 사이드카 프록시로 옮겨서 실행
